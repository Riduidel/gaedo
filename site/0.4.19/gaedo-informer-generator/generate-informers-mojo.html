gaedo-informer-generator:generate-informers

Full name:

com.dooapp:gaedo-informer-generator:0.4.19:generate-informers

Description:

Goal generating informer classes in a target folder from bean classes.

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: generate-sources.

Optional Parameters

Name Type Since Description
enums String[] - A list of qualified names of enums containing the enums not in this project source path, but required to be enums.
excludes String[] - Defines which of the included files in the source directories to exclude (none by default).
generateMappings boolean - Flag indicating if informer mappings file should be generated
Default value is: false.
includeStaticProperties boolean - Define if informers should be generated for static fields. As it's not really interesting to have such a behaviour, it is disabled as default.
Default value is: false.
includes String[] - Sources that will be scanned
informerMappings File - Output file where those classes will be written
Default value is: ${project.build.directory}/classes/META-INF/informer-mappings.properties.
output File - Output file where those classes will be written
Default value is: ${project.build.directory}/generated/informers.
propertiesExcludes String[] - A list of properties names that should be excluded. Format of properties written here can be in two forms:
  • Simple strings
  • Javadoc-comaptible properties : package.class#property
requiredAnnotations String[] - Only classes having those annotations defined wil have their informers generated Annotation name should be fully qualified.
requiredInterfaces String[] - Only classes implementing those interfaces will have their informers generated. Interface name should be fully qualified. Notice that this array is by default empty, which make it require no interface at all.
